# Host Agent Reuse for Developer Preview

The developer-preview workflow will prioritize Host Agent Reuse so users can run their existing coding agents with explicitly exposed configuration, MCP servers, skills, and authentication setup. This lowers isolation assurance compared with a more isolated Development Environment, but avoids making users recreate the personalized agent environment that makes coding agents useful in practice; stricter execution through a User-Provided Environment or future isobox-Managed Environment remains explicit rather than the default preview path.
